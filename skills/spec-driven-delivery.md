# Skill: Spec-Driven Delivery

Use this workflow for all meaningful feature work.

1. Write a feature spec in `specs/drafts/`.
2. Write a test plan that names the exact checks required to accept the feature.
3. Ask for approval before implementation.
4. Move the pair into `specs/approved/` once approved.
5. Implement only approved behavior.
6. If scope changes, update the spec before continuing.
